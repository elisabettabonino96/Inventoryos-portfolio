About this project

Portfolio Edition — July 2026

InventoryOS started as a real internal tool: a small food & beverage distributor was tracking inventory across manual processes and disconnected spreadsheets, with no easy way to see stock levels or know what needed reordering. This portfolio edition rebuilds that original product concept and workflow with anonymized branding and a fictional dataset, so it can be shared publicly.

Note on this demo: the original production build reads and writes live inventory data through the Google Sheets API. This portfolio edition replaces that live data source with a generated fictional dataset (and saves adjustments to this browser only) so it can be shared publicly without exposing real credentials or business data.

Problem

Small distributors often manage inventory through spreadsheets and disconnected systems.

Solution

Designed and built a lightweight inventory management application that provides inventory visibility, search, filtering, user management, and operational dashboards.

Project Status

Current capabilities: inventory visibility, search and filtering, user roles and permissions, operational dashboards, stock monitoring, and inventory sorting.

Planned roadmap: transaction history, inventory movement tracking, purchase order management, supplier management, reporting and analytics enhancements, reorder point management and low-stock alerts, and direct QuickBooks integration for automatic inventory synchronization.

This portfolio version represents the original product vision and selected functionality. The production version continues to evolve based on user needs and business requirements.
