# InventoryOS — Portfolio Edition

**Live demo:** _add your GitHub Pages link here once enabled (Settings → Pages)_

A lightweight inventory management system, designed and built for a small food & beverage distributor that was tracking stock through manual processes and disconnected spreadsheets. This portfolio edition is anonymized and runs on a generated fictional dataset so it can be shared publicly without exposing real credentials or business data — see the "About Project" panel inside the app for the full case study.

## Try it

Open `index.html` in any browser (or use the live demo link above). Select **Warehouse Operator** for the day-to-day inventory view, or **Administrator** for the full dashboard, stock adjustments, and roadmap.

## Problem

Small distributors often manage inventory through spreadsheets and disconnected systems, with no easy way to see stock levels at a glance or know what needs reordering.

## Solution

Designed and built a lightweight inventory management application that provides inventory visibility, search, filtering, user management, and operational dashboards.

## My role

Product strategy, requirements gathering, workflow design, UX design, front-end implementation, and Google Sheets integration (production build). Responsible for defining product requirements, designing operational workflows, and translating business needs into a working software solution.

## Technology

HTML, CSS, JavaScript — no framework or build step. The original production build reads and writes live inventory data through the Google Sheets API; this portfolio edition swaps that for a generated dataset (with adjustments saved to `localStorage` in your browser) so it can run standalone.

## Roadmap

Current: dashboard, search, filters, user roles. Planned: transaction history and movement tracking, purchase orders and supplier management. Later stage: direct QuickBooks integration for automatic inventory synchronization. Roadmap priorities are based on user feedback and operational requirements.
